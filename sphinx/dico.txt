Composant algorithmique
Le composant algorithmique est un sous-ensemble cohérent de code de prévision numérique susceptible de faire l'objet d'une seule et unique description logique et dont le comportement est régi par un ensemble structuré d'informations ;
Cet ensemble d'informations peut prendre la forme d'une namelist, ou d'un « fichier de configuration » de format quelconque? mais bien plus probablement un XML quelconque.
Configuration
Une configuration est un enchaînement logique de tâches visant à satisfaire un besoin donné en terme d'expérimentation numérique. Elle définit complètement la séquence d'enchaînement, les dépendances éventuelles entre tâches, le flux de données.
Données d'entrée
Les ressources mobilisées dans un flux de données et dont la présence, sous une forme appropriée est exigée pour un traitement par un noyau exécutable constituent les données d'entrée d'une tâche ou d'un groupe de tâches.
Données de sortie
Les ressources produites dans un flux de données sous une forme particulière après la mise en ?uvre d'un noyau exécutable constituent les données de sortie d'une tâche. Ces données peuvent ou non, devenir les données d'entrée d'une autre tâche ou de la même tâche dans le cas d'une exécution cyclique.
Expérimentation
L'expérimentation est le déploiement dans un espace de temps ou sur un jeu limité de ressources d'une configuration. Cette spécification est souvent opéré via des paramétrisations.
Mode d'exécution
Le mode d'exécution est l'ensemble des modalités qui caractérisent une mise en ?uvre spécifique d'un noyau exécutable. Cela recouvre notamment, mais peut-être pas exclusivement : la mise en forme ou la conversion de données d'entrée, la définition de la ligne de commande, le positionnement de l'environnement informatique, des possibilités de cyclage de l'exécution, etc. ;
Remarque : certaines de ces modalités peuvent n'avoir de sens que dans l'espace physique.
Par ailleurs, en conjonction avec les qualités propres à un noyau d'exécution, le mode d'exécution précise les types de ressources autorisées en entrées ou potentiellement présentes en sortie d'exécution (on parle alors d'enveloppe de ressources).
Noyau exécutable
Un noyau exécutable est une entité qui joue un rôle actif dans le flux de données d'une simulation numérique. C'est souvent un exécutable binaire de code compilé constitué par l'agrégation d'un ou plusieurs noyaux algorithmiques, mais pas toujours. Cela peut être aussi une combinaison de plusieurs binaires ou même un « simple » script interprété.
Dans le cadre des sciences environnementales, le modèle de prévision numérique est le noyau d'exécution par excellence.
Paramétrisation
Une paramétrisation est la méthode par laquelle des éléments variables, ou paramètres, d'une configuration peuvent être définis. Ces éléments peuvent faire l'objet de lexiques. On peut dire de ces paramètres qu'ils jouent le rôle des variables dans un programme, dont ils partagent a minima les règles de portée, mais sans limiter a priori le type d'information qu'ils véhiculent, qui peut elle-même être complexe.
Ressource
La ressource est l'entité de base manipulable au cours d'une expérimentation de prévision numérique et ayant un sens du point de vue de cette expérimentation. C'est l'objet météorologique par excellence. Par exemple une analyse, l'exécutable binaire d'un modèle, un champ de pression de surface, etc., sont des ressources.
A contrario, un répertoire d'un système de fichiers, un calculateur, etc. ne sont pas des ressources.
Séquence
La séquence est le regroupement ordonné de données d'entrée, d'un noyau exécutable associé à un mode d'exécution particulier, et de la collecte et du traitement éventuel de données produites ou données de sortie.
Tâche
La tâche est une séquence logique spécifiquement paramétrée qui regroupe ainsi l'obtention de données d'entrée, la mise en ?uvre d'un noyau exécutable suivant un mode d'exécution particulier, la collecte et le traitement éventuel de données produites ou données de sortie.

Contexte d'exécution
Le contexte d'exécution est le cadre réel dans lequel se déroule une tâche ou plusieurs tâches dans le cas d'un système couplé. Il définit notamment la portée des ressources manipulées ou des paramètres d'environnement modifiés.
Environnement
L'environnement est l'ensemble des conditions externes à la configuration dans lesquelles se réalise une expérimentation numérique. Tout ce qui relève de l'implémentation informatique contribue ainsi à définir un environnement donné. Les variables d'environnement, sont les éléments paramétrables qui permettent de modifier cet environnement. Elles peuvent représenter n'importe quelle entité logique et pas seulement des valeurs « scalaires » (comme le sont les variables shell par exemple).
Pilote
Le pilote d'exécution réalise le séquencement physique effectif d'une configuration ou d'un sous-ensemble de la configuration. Il opère en particulier la coordination des différents contextes d'exécutions et définit les diverses responsabilités entre ceux-ci ainsi que ce qui peut-être factorisé et regroupé par ses soins.
Système couplé
Un système couplé permet la mise en ?uvre simultanée et coordonnée de plusieurs noyaux exécutables, définis au sein d'une même tâche ou dans des tâches distinctes pour exprimer plus précisément le flux de données. Le caractère couplé du système est assuré dans la mise en pratique de cet ensemble dans un contexte d'exécution donné ou par synchronisation de plusieurs contextes d'exécution.
